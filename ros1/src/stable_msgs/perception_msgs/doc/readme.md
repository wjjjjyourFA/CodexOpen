### 概述
### 🚗 背景认知：感知模块常包含

- **静态环境感知**（栅格地图、占据图等）
- **动态物体感知**（车、人、非机动车等目标）
- **车道线/交通元素识别**（车道线、红绿灯等）
- **融合后的世界模型**（Entity Map）



### ✅ 应该绑定冻结坐标系的典型数据：

| 数据类型                       | 是否需冻结坐标系 | 理由                                           |
| ------------------------------ | ---------------- | ---------------------------------------------- |
| `DetectedObject` 检测目标      | ✅ 是             | 多用于异步跟踪/预测/融合，不能因坐标变动而错位 |
| `LaneDetection` 车道线         | ✅ 是             | 车道线用于局部地图、规划，需冻结在感知时刻     |
| `TrafficLightDetection` 红绿灯 | ✅ 是             | 需要与规划时刻匹配状态，且位置不能随车体漂移   |
| `PerceptionEntityMap`          | ✅ 是             | 综合地图、静态障碍，必须冻结一致场景快照       |
| `ObjectPoint` 或轨迹点         | ✅ 是             | 跟踪轨迹中每个点都需在历史时刻坐标下可重建     |



## 🔚 总结一句话：

> **凡是后续模块（如跟踪、融合、预测）需要依赖空间一致性重现的感知结果，都需要绑定冻结坐标系**，否则数据在空间上将“漂移不定”。



### EntityMap.msg

用于融合后的世界模型，包含静态栅格图 + 动态目标 + 交通元素等，用于传达关于多个检测或追踪实体（如行人、车辆、障碍物等）的整体信息。这些信息可能包括实体的位置、速度、形状、类别以及其他属性。

EntityMap.msg 通常包含一个实体列表，每个实体都有自己的属性和特征，以便系统进行进一步的处理和分析。





## ✅ 总结对比：



| 对比项       | `ClassifiedObject`         | `SemanticObject`                          |
| ------------ | -------------------------- | ----------------------------------------- |
| 含义         | 已分类目标（检测、跟踪等） | 具备语义的场景元素（如地面元素）          |
| 类别粒度     | 通常是物体类别，如 "car"   | 可是语义标签，如 "crosswalk"、"freespace" |
| 场景层次     | 感知层                     | 地图/决策层，或高阶理解                   |
| 是否动静均可 | ✅                          | 多为静态，但也可标记动态场景语义          |
