

初始传感器信息，不用带上冻结坐标系
但传感器的融合信息，应当带上冻结坐标系
换言之，*融合传感器信息*，是*冻结坐标系*的起始点
而 *融合传感器信息*，这一步，实际的发生在*传感器融合模块*中，而不是在*传感器模块*中


### ❌ 不需要冻结坐标系的数据：



| 数据类型                   | 是否需冻结坐标系     | 理由                                         |
| -------------------------- | -------------------- | -------------------------------------------- |
| `IMU / GPS 原始数据`       | ❌ 否                 | 实时使用、不断更新，不用于异步模块融合       |
| `当前车辆位置（Odometry）` | ❌ 否                 | 实时绑定车体坐标，不做历史对齐               |
| `车控反馈 / 状态数据`      | ❌ 否                 | 时效性要求高、不需要追溯                     |
| `传感器原始点云 / 图像`    | ❌ 否（但需要时间戳） | 点云一般以 sensor_frame 发布，不绑定冻结坐标 |



“INS” 是 **惯性导航系统**（**Inertial Navigation System**）的缩写。

------

#### ✅ 简单解释

**INS** 是一种利用 **加速度计（accelerometer）** 和 **陀螺仪（gyroscope）** 来估计位置、速度和方向的系统。

它**不依赖外部信号**（如 GPS），可以在没有通信或卫星信号的情况下进行导航。

------

#### 🧠 INS 的核心原理：

1. **陀螺仪**：测量角速度，用来估计方向（姿态）。
2. **加速度计**：测量线性加速度。
3. **积分计算**：
   - 对加速度积分得到速度；
   - 对速度积分得到位置。
