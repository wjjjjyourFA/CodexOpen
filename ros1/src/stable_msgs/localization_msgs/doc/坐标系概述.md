## 🗺️ 一、系统中常见的定位方式与坐标系对照



| 定位来源     | 是否全局 | 常用坐标系         | 是否绝对 | 是否易漂移  | 典型用途                 |
| ------------ | -------- | ------------------ | -------- | ----------- | ------------------------ |
| ✅ GPS / GNSS | ✅ 是     | 高斯 / WGS84 / UTM | ✅ 是     | ❌ 否        | 绝对定位、导航、地图配准 |
| ✅ RTK-GPS    | ✅ 是     | 高斯 / WGS84 / UTM | ✅ 是     | ❌ 否        | 精确全局定位（厘米级）   |
| ✅ GNSS + IMU | ✅ 是     | 地图坐标 + 航向    | ✅ 是     | ❌（可补偿） | 室外高精度定位           |
| ✅ 多源融合   | ✅ 是     | 地图系             | ✅ 是     | ❌           | 自动驾驶高精定位模块     |
| 🔁 SLAM       | ❌ 否     | local / odom       | ❌ 否     | ✅ 是        | 室内导航、重定位         |
| 🔁 Lidar-Odom | ❌ 否     | odom               | ❌ 否     | ✅ 是        | 近距离跟踪、轨迹预测     |
| 🔁 轮速编码器 | ❌ 否     | odom               | ❌ 否     | ✅ 高        | 简易定位、辅助传感器     |

------

## ✅ 分类总结

### ✅ 全局定位（Global Coordinate）

- 来源：GPS、RTK、GNSS/INS、多源融合
- 坐标系：WGS-84、UTM、高斯坐标、地图坐标（map）
- 特征：**不随车辆移动、绝对位置**

### 🔁 局部定位（Local Coordinate）

- 来源：SLAM、轮速里程计、视觉惯导、Lidar odom
- 坐标系：odom、local_pose（freezed）、base_link
- 特征：**原点随时间可漂移**（odom）或 **设定不变原点**（local）

------

## 📌 你可以这样理解：



| 你说的名称             | 实际含义                                   |
| ---------------------- | ------------------------------------------ |
| 高斯坐标系             | 是 **GPS坐标系的一种地理投影表达**（全局） |
| GPS 定位模块           | 得到的是 **高斯/WGS-84 全局位置**          |
| ODOM 定位（轮速/SLAM） | 得到的是 **局部位置**（相对）              |
| SLAM                   | 类似 odom，但基于传感器融合                |

------

## 🔍 所以你整个系统可能是这样：

```
                   ┌────────────┐
                   │ GPS 模块   │─────> 高斯坐标（global_pose）
                   └────────────┘
                         ▲
                         │
            ┌────────────────────────────┐
            │ 多传感器融合（IMU + Lidar + GPS） │────> map_pose（融合定位）
            └────────────────────────────┘
                         │
                         ▼
               ┌──────────────────┐
               │ 感知模块 / 控制模块 │
               └──────────────────┘
                         ▲
                         │
              ┌────────────────────────┐
              │  SLAM、轮速编码器、Lidar Odom │────> odom_pose / local_pose
              └────────────────────────┘
```

------

## ✨ 总结建议

- ✅ **高斯坐标 ≈ GPS 定位输出**，属于全局坐标系；
- ✅ 可以叫它 `global_pose_gauss` 或 `gps_pose`；
- ✅ 其他如 SLAM、轮速、IMU 得到的是相对局部坐标；
- ❗注意 SLAM 也可“重定位”到全局地图坐标，但那需要地图对齐。
